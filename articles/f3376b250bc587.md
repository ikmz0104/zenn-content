## 書き方
参考程度に～

:::よく使うソース
```
## コードの開始と終了の合図
@startuml
処理
@enduml

## 人間やデータベースは定義できる
actor: ユーザＡ
database: テーブルＡ

## リクエスト・レスポンス
ユーザA -> 画面A : リクエスト内容  //ユーザAが画面Aにリクエスト内容に応じた処理を行う
API --> ユーザA : レスポンス内容  //APIからユーザAにレスポンス内容に応じた処理が投げられる

## フローの開始・終了点の可視化
activate
desactivate

## 特定フローへのグリッド＆コメント
par
! ここに処理を書く、ここの処理がグリッドされる !
note right : コメント
end
```
:::
